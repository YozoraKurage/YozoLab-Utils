# PlayMode Pose Baker

シーン再生中のアバターのボーンTransform（PhysBoneなどの揺れ物が動いた後の状態）を
キャプチャしてAnimationClipとして書き出し、そのクリップ（または任意のAnimationClip）を
Editモードのアバターに焼き込むエディタ拡張です。
揺れ物が「落ち着いた」状態をシーン再生前のアバターでプレビューしたい場合に使います。

## 使い方

1. `YozoLab > PlayMode Pose Baker` でウィンドウを開き、アバタールートを指定する
2. シーンを再生し、PhysBoneが安定するまで待つ
3. ウィンドウの「現在のポーズをキャプチャ」を押す
4. 再生を終了し、「AnimationClipとして書き出し...」でクリップ化する
   （書き出したクリップは自動で焼き込み欄にセットされます）
5. 「AnimationClipをシーンへ焼き込み」を押す

焼き込みはUndo（Ctrl+Z）で元に戻せます。

## AnimationClipの書き出し

「AnimationClipとして書き出し...」で、キャプチャしたポーズを
1キーのAnimationClipアセット（.anim）として保存できます。
位置/回転/スケールのチェックが書き出し対象のプロパティになります。
生成されるクリップ名は必ず末尾が `_generic` になります
（Humanoidボーンを含まないGenericなクリップであることを示すため）。

- **Humanoidマッピング対象のボーン（指・目・顎含む）のキーは書き出されません**。
  Humanoidボーンへの直接のTransformカーブはマッスル制御と競合するためで、
  素体のポーズはHumanoidクリップ側で扱ってください（除外数は書き出し時に表示されます）。
- パスはアバタールート基準なので、アバターのAnimator（FX/Gestureレイヤーや
  Animationウィンドウでのプレビュー）からそのまま参照できます。
- 同じパスに上書き保存した場合、GUIDは維持されるため既存の参照は壊れません。
- アニメーションのバインディングパスでは同名の兄弟Transformを区別できないため、
  2番目以降の同名兄弟はスキップされます（書き出し時に件数が表示されます）。

## AnimationClipの焼き込み

「焼き込み（AnimationClip → シーン）」では、書き出したポーズのクリップに限らず、
**任意のAnimationClip**を指定してそのポーズをシーン上のアバターに焼き込めます。
サンプル時刻のスライダーで、クリップ内のどの時点のポーズを使うか選べます。

- AnimationMode（Animationウィンドウのプレビューと同じ仕組み）でサンプリングするため、
  **HumanoidクリップもGenericクリップも同じように処理されます**。
  HumanoidクリップはAnimatorのAvatar経由でマッスルカーブがボーン回転に解決されます
  （アバタールートにHumanoid設定済みのAnimatorが必要です）。
- 一時的なサンプリング結果からTransform（位置/回転/スケール）だけを差分適用するため、
  ブレンドシェイプやマテリアル等のプロパティは焼き込まれません。
- アバタールート自体は動かしません（ルートモーションは持ち込まれません）。

## キャプチャ対象

- **PhysBoneの影響ボーンのみ**（デフォルト）:
  各VRCPhysBoneのRoot Transform配下（Ignore Transforms除外）のみをキャプチャします。
  アニメーションによる素体のポーズを持ち込まず、揺れ物だけを焼き込みたい場合はこちら。
- **全Transform**: アバター配下の全Transformをキャプチャします。
  再生中のアニメーションのポーズごと焼き込まれる点に注意してください。

## 仕組みとNDMF環境での注意点

- NDMF（Apply on Play）はビルド前後でGlobalObjectIdが安定しないため、
  本ツールは**アバタールートからの相対階層パス**でボーンを照合します。
  Armature配下のボーン階層はビルド前後で基本的に維持されるため、この方式で対応できます。
- NDMF導入環境では、さらに**NDMFのObjectRegistryを使った照合**が自動で有効になります。
  Resolvingフェーズ（各ツールによる変形の前）に本ツールのプラグインが
  全ボーンのObjectReferenceを作成して「ビルド前のパス」を確定させ、
  キャプチャ時にビルド後のボーンからビルド前のパスを逆引きします。
  これにより、Modular AvatarのMerge Armatureなどで**ビルド中に移動・置換された
  ボーンも、Editモードの正しい階層パスでAnimationClipに書き出されます**。
  NDMF未導入でもコンパイル・動作し、その場合は階層パス照合のみになります。
- キャプチャ結果は `Library/YozoLab/PoseBaker/LastCapture.json` に保存され、
  再生終了時のドメインリロードを跨いで保持されます（バージョン管理には含まれません）。
- NDMFがビルド時に生成したオブジェクトはEditモードに存在しないため、
  書き出したクリップを焼き込んでも反映されません（バインド先がないため無視されます）。
- アバタールート自体のTransform（再生中の移動位置）は持ち込みません。

## 制限

- VRC SDKへのアセンブリ参照を持たないため、PhysBoneの検出はリフレクションで行っています。
  PhysBone以外の揺れ物（Cloth等）を焼き込みたい場合は「全Transform」を使ってください。
- 書き出したクリップにはHumanoidボーンのキーが含まれないため、焼き込んでも
  素体のポーズ（Tポーズ）は崩れません。一方、任意のHumanoidクリップを焼き込んだ場合は
  素体のポーズも変わります。必要に応じてUndoで戻してください。
